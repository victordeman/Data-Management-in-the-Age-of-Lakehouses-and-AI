Week 1-2: Why Traditional Data Management Broke & What Replaced It

From RDBMS → NoSQL → data lakes → lakehouse paradigm

Lecture Overview

This week explores the evolution of data management systems, the challenges of traditional approaches, and the emergence of modern paradigms. We'll revisit the 5 Vs of Big Data and dive into chapters 1-3 of Kleppmann's "Designing Data-Intensive Applications."

  • Understand why RDBMS struggled with scale and variety.
  • Learn about NoSQL's trade-offs and data lakes' flexibility.
  • Discover how lakehouses combine the best of lakes and warehouses.

Evolution Timeline

A visual overview of data management progression:

RDBMS Era (1970s-2000s)

Structured data with SQL, ACID guarantees, but limited scalability for big data.

NoSQL Rise (2000s)

Flexible schemas, horizontal scaling, but often sacrificed consistency.

Data Lakes (2010s)

Store raw data cheaply, but lacked governance and query performance.

Lakehouse Paradigm (2020s+)

Open formats like Delta Lake add transactions, schema enforcement, and AI readiness to lakes.

The 5 Vs of Big Data Revisited

Volume

Handling massive data scales beyond traditional RDBMS.

Velocity

Real-time processing demands.

Variety

Structured, semi-structured, unstructured data.

Veracity

Data quality and trustworthiness.

Value

Extracting insights efficiently.

Key Concepts from Kleppmann Ch. 1-3

Summary of "Designing Data-Intensive Applications" chapters:

  • Ch. 1: Reliability, scalability, maintainability as core goals.
  • Ch. 2: Data models (relational vs. document vs. graph) and query languages.
  • Ch. 3: Storage engines (log-structured vs. page-oriented) and indexing.

Key takeaway: Choose data models based on application needs, not hype.

Linux Data Toolbox

Essential tools for a Linux-first environment:

Bash

Scripting for automation.

#!/bin/bash
echo "Hello, Data World!"

tmux

Session management.

tmux new -s data-session

jq

JSON processing.

echo '{"key": "value"}' | jq '.key'

Docker

Containerization.

docker run -it ubuntu bash

systemd

Service management.

systemctl status

Hands-on: Lab 0 - Personal Linux Data Workstation Setup

Set up your environment:

  1. Install Ubuntu or a Linux distro.
  2. Update packages: sudo apt update && sudo apt upgrade
  3. Install tools: sudo apt install tmux jq docker.io
  4. Enable Docker: sudo systemctl start docker
  5. Test: Run a container and process sample data.
